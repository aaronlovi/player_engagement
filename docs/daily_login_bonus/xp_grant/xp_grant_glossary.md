# Daily Login Bonus – XP Grant

**Section:** Glossary

**Purpose:** Define terminology used in the Business Requirements and subsequent technical documentation for the Daily Login XP Grant system.

| **Term**                      | **Definition**                                                                                                     | **Notes / Context**                                                                                                          |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| **XP (Experience Points)**    | A numerical measure of player progress earned through gameplay or rewards.                                         | Used to advance player levels or unlock content.                                                                             |
| **Daily Login Bonus**         | A reward granted to players for logging into the game once per day.                                                | Can include XP, currency, or items; this design focuses on XP.                                                               |
| **Streak**                    | The count of consecutive days a player successfully claims their daily reward.                                     | Reset, decayed, or capped depending on policy configuration.                                                                 |
| **Streak Continuation Model** | The rule set determining how streak XP progression behaves over time.                                              | Configurable models include Plateau/Cap, Weekly Cycle Reset, Decay Curve, Tiered Seasonal Reset, and Meta-Reward Milestones. |
| **Grace Period**              | A configurable tolerance that allows a streak to continue even if a player misses one or more days (within limit). | E.g., 1-day grace means missing one day does not reset streak.                                                               |
| **Claim Window**              | The time range each day when the player can claim their daily reward.                                              | Typically defined per player timezone or server policy (e.g., 00:00–05:00 local).                                            |
| **Reward Day**                | The logical day boundary used to determine claim eligibility.                                                      | Calculated server-side based on user anchor timezone.                                                                        |
| **Anchor Timezone**           | The primary timezone associated with a user’s account for daily rewards.                                           | Prevents double-claiming when traveling between timezones.                                                                   |
| **XP Curve**                  | The table or formula that defines how XP increases with streak length.                                             | E.g., Day1: +0%, Day2: +10%, Day7: +100%.                                                                                    |
| **Policy Version**            | Version identifier of the configuration defining XP grant rules.                                                   | Allows auditability and rollback of reward logic.                                                                            |
| **Operator Console**          | Administrative interface where live-ops teams configure XP grant parameters.                                       | Supports editing of base XP, streaks, grace, multipliers, and claim window.                                                  |
| **Segment (Player Segment)**  | A group of players defined by behavior, lifecycle, or demographics.                                                | Used to apply different reward policies (e.g., new vs returning players).                                                    |
| **Event / Season**            | A limited-time campaign or game season that may modify XP rewards.                                                 | Seasonal boosts may apply multipliers to XP grants.                                                                          |
| **XP Ledger**                 | The system record tracking all XP transactions and their reasons.                                                  | Each grant is logged with correlation ID and policy version.                                                                 |
| **Receipt ID**                | A unique identifier returned to the client confirming a successful XP claim.                                       | Used for idempotency; retries return same receipt.                                                                           |
| **Idempotency**               | Ensuring repeated requests for the same action do not create duplicate results.                                    | Guarantees only one XP grant per reward day.                                                                                 |
| **Anti-Abuse Controls**       | Systems or policies that prevent exploitation of XP grants.                                                        | Include device limits, IP checks, and automation detection.                                                                  |
| **Progression System**        | The overall system that manages player levels and advancement.                                                     | XP from daily login contributes to this system.                                                                              |
| **Visual Feedback**           | Immediate UI response when XP is awarded.                                                                          | Typically an animation, toast, or level bar update.                                                                          |
| **Fair Play**                 | Design principle ensuring rewards don’t create gameplay imbalance.                                                 | XP should not provide competitive advantage in PvP.                                                                          |
